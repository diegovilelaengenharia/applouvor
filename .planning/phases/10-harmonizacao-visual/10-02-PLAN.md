<objective>
Swap rigid `font-size` literals and non-standard `font-weight` values for canonical `--font-size-*` / `--font-weight-*` tokens across the highest-traffic typography surfaces: the two most-reused component files (pib-cards, page-sub-header), the timeline-cards component, and the five highest-traffic page files (escalas, lider, oracao, devocionais, repertorio).

Depends on Plan 01 — the token cascade must be clean first (a hijacked --primary or --radius-lg would distort how these tokens read).

Purpose: A single typographic scale (16px body / 14px secondary / 20px heading / 24px display) applied via tokens, so nothing is oversized or squeezed on mobile.
Output: Eight files where every in-scope `font-size` references a `--font-size-*` token and every `font-weight: 750/900` is replaced with `var(--font-weight-bold)` (KPI 800 kept as documented exception).
</objective>
