# Phase 10 — UI Design Contract: Harmonização Visual

> Visual and interaction contract for harmonising typography, spacing, button system, and card density across all admin/ and app/ pages. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — existing `assets/css/design-system.css` is the single source of truth |
| Preset | not applicable |
| Component library | none (Vanilla CSS + PHP) |
| Icon library | Lucide (inline SVG, already used throughout) |
| Font | Inter (body) · Inter Tight (headings/display) via Google Fonts — already loaded in design-system.css |

**Source confirmation:** `design-system.css` declares `--font-body`, `--font-heading`, `--font-display` and the full spacing/color token set. All declarations in this spec reference those existing var() names. No new framework or library is introduced.

## Spacing Scale

Canonical 8-point scale already declared in `design-system.css`. This phase enforces use of these tokens — no page-level magic numbers.

| Token | CSS Var | px Value | Usage |
|-------|---------|----------|-------|
| xs | `--space-1` | 4px | Icon-to-label gaps, badge inner padding |
| sm | `--space-2` | 8px | Inline element gaps, compact button padding |
| md | `--space-4` | 16px | Default card padding, form field padding |
| lg | `--space-6` | 24px | Section gaps, page content padding |
| xl | `--space-8` | 32px | Major section breaks |
| 2xl | `--space-12` | 48px | Hero/header sections |
| nav-bottom | `--bottom-nav-height` | 70px | Scroll bottom offset on mobile |

**Exceptions:**
- Touch target minimum: 44×44px (`min-height: 44px` on all interactive elements — already enforced by `.btn` in buttons.css but must be audited for custom page buttons)
- `page-sub-header-inner` height: 48px (pre-existing, locked as canonical)
- Card border-left accent strip: 4–5px (visual, not spacing token)
- Avatar stack overlap margin: −8px (negative, intentional overlap pattern)

**Canonical page padding:** All `.app-content` children must use `padding: var(--space-6)` (24px) as the page-level horizontal pad. The `page-sub-header` already uses negative margin to compensate (`margin-left: -24px; margin-right: -24px`). This pattern is locked.

## Typography

Four canonical roles. All sizes bind to existing `--font-size-*` tokens in `design-system.css`.

| Role | CSS Var | px | Weight Token | Line Height | Font Family | Usage |
|------|---------|-----|-------------|-------------|-------------|-------|
| Body | `--font-size-base` | 16px | `--font-weight-normal` (400) | `--line-height-normal` (1.5) | `--font-body` (Inter) | Paragraphs, descriptions, form labels, list items |
| Secondary / Label | `--font-size-sm` | 14px | `--font-weight-medium` (500) | `--line-height-normal` (1.5) | `--font-body` (Inter) | Metadata, badges, helper text, table cells, nav labels |
| Heading (page/section) | `--font-size-xl` | 20px | `--font-weight-semibold` (600) | `--line-height-tight` (1.25) | `--font-heading` (Inter Tight) | Page titles in header, section headings, card titles |
| Display (hero/KPI) | `--font-size-2xl` | 24px | `--font-weight-bold` (700) | `--line-height-tight` (1.25) | `--font-display` (Inter Tight) | KPI values, dashboard hero numbers, date-box day |

**Out-of-scale uses (allowed exceptions):**
- `--font-size-xs` (12px) for caption-level only: badge labels, avatar initials, date-box month labels, bottom-nav labels. Must use `--font-weight-semibold` (600) for legibility at small size.
- `--font-size-lg` (18px) for sub-page section titles when a step below Display is needed (e.g., form section headers).
- KPI numbers on dashboard (`kpi-value`) may use `--font-size-3xl` (30px) with weight 800 — these are intentional display exceptions, not body text.

**Prohibited:**
- Bare `font-size` px or rem literals in page CSS where a `--font-size-*` token exists. Replace with the token.
- `font-size: 1.05rem`, `font-size: 0.78rem`, `font-size: 1.7rem` etc. — these arbitrary intermediate sizes found in page CSS must be rounded to the nearest canonical token.
- `font-weight: 750` and `font-weight: 900` in page CSS — not valid across all rendering engines; replace with `--font-weight-bold` (700) or `--font-weight-semibold` (600).

## Color

60/30/10 split + semantic colors. All values reference existing design-system.css tokens.

| Role | CSS Var | Hex | Usage |
|------|---------|-----|-------|
| Dominant 60% — App background | `--bg-body` / `--bg-app` | `#f8fafc` (slate-50) | Page background, scroll area, body |
| Secondary 30% — Surfaces | `--bg-surface` | `#ffffff` | Cards, panels, modals, header, sidebar |
| Secondary alt — Inset/alt surfaces | `--bg-surface-alt` | `#f1f5f9` (slate-100) | Tab backgrounds, segmented controls, sticky headers |
| Accent 10% — Primary interactive | `--primary` / `--color-primary` | `#3b82f6` (blue-500) | See accent reserved list below |
| CTA — Action accent | `--color-accent` | `#FFC501` (yellow-500) | Explicit CTA only — "Confirmar presença" primary action, "Nova escala" FAB |
| Destructive | `--color-danger` | `#ef4444` (red-500) | Delete actions, absent/declined status, destructive confirms only |
| Success semantic | `--color-success` | `#22c55e` (green-500) | Confirmed status badges, streaks, "orando" state |
| Warning semantic | `--color-warning` | `#f59e0b` (amber-500) | Pending status, pastoral alerts, late/urgente flags |

**Accent (`--primary` blue-500) reserved for:**
- Active tab/segment indicator
- Focus ring outline
- `.btn-primary` background
- Border highlight on `.pib-card:hover`
- Active `.bottom-nav-item` icon and label
- `.radio-pill.active` and `.filter-tab.active` (when primary-themed)
- Confirmation badge ("X/Y confirmados") when `some-confirmed`
- Page sub-header link/action items

**CTA orange (`--color-accent` / `--orange-500`) reserved for:**
- "Confirmar presença" footer button on escala_detalhe (músico)
- FAB create buttons on devocional/oracao pages
- "Registrar faltas" action button
- Badge: sugestões pendentes (admin dashboard)

**Green (`--color-success`) reserved for:**
- Confirmed status badges
- Streak flame/icon
- Answered prayer state
- "all-confirmed" badge

**Semantic reuse rule:** `devocionais.css` and `oracao.css` currently redefine `--primary` in `:root` to green (#22c55e / #16a34a). This is the most dangerous pattern in the codebase — it silently overrides the blue primary for ALL subsequent rules in the cascade. **These `:root` blocks must be removed** and replaced with page-scoped custom properties or semantic color tokens (`--color-spiritual`, `--reading-primary`) that already exist in design-system.css. The accent color for spiritual pages is `--color-espiritual` (`--cyan-500`) or `--reading-primary` (`--green-600`), not a `:root` primary override.

**Dark mode:** All tokens already have dark-mode overrides in `body.dark-mode {}` block in design-system.css. Page CSS must not redeclare dark-mode color values as raw hex — reference the semantic tokens and dark mode handles them automatically.

## Button System Contract

The `components/buttons.css` is already well-formed. This spec locks the matrix so page-level overrides are forbidden.

### Size Matrix

| Class | Height | Padding H | Padding V | Font Size | Border Radius | Touch Safe |
|-------|--------|-----------|-----------|-----------|---------------|------------|
| `.btn` (default) | 44px min | 20px | 12px | 16px (`--font-size-base`) | `--radius-lg` (12px) | Yes |
| `.btn-sm` | 36px min | 16px | 8px | 14px (`--font-size-sm`) | `--radius-md` (8px) | No (not primary actions) |
| `.btn-lg` | 52px min | 24px | 16px | 18px (`--font-size-lg`) | `--radius-xl` (16px) | Yes |
| `.btn-icon` | 44×44px | 0 | 0 | — | `--radius-lg` (12px) | Yes |
| `.btn-icon-sm` | 36×36px | 0 | 0 | — | `--radius-md` (8px) | No |

**Rule:** Any button that is a primary user action (confirm, save, create, submit) must use `.btn` or `.btn-lg` (never `.btn-sm`) to guarantee 44px touch target.

### Variant Hierarchy

| Class | Background | Color | Border | When to Use |
|-------|-----------|-------|--------|-------------|
| `.btn-primary` | `--color-primary` | white | none | Single most important action per view |
| `.btn-accent` | `--color-accent` (yellow) | white | none | Músico's confirmation CTA only |
| `.btn-secondary` | `--bg-surface` | `--color-text` | `--color-border-medium` | Secondary actions (edit, cancel, back) |
| `.btn-ghost` | transparent | `--text-secondary` | none | Tertiary actions, icon-only row actions |
| `.btn-danger` | `--color-danger` | white | none | Destructive actions only |
| `.btn-success` | `--color-success` | white | none | Confirm-received / mark-as-answered |

### States

| State | CSS | Effect |
|-------|-----|--------|
| Default | — | As declared |
| Hover | `translateY(-1px)` + `--shadow-md` | Lift effect |
| Active/Press | `scale(0.97)` | Tactile confirmation |
| Disabled | `opacity: 0.5; cursor: not-allowed` | No hover effect |
| Loading | `.btn-loading` class | Spinner overlay, text transparent |
| Focus-visible | `outline: 3px solid --primary; outline-offset: 2px` | WCAG AA |

**Prohibited page-level button patterns to remove:**
- `.pray-btn` in oracao.css: replace with `.btn .btn-primary` / `.btn .btn-ghost`
- `.btn-new-prayer` in oracao.css: replace with `.btn .btn-primary .btn-block`
- `.btn-hero-confirm`, `.btn-hero-decline` in pib-cards.css: replace with `.btn .btn-success` / `.btn .btn-secondary`
- `.btn-print` in relatorios.css: replace with `.btn .btn-secondary`
- `.btn-registrar-faltas` in escalas.css: replace with `.btn .btn-sm` + orange variant (or keep as-is since it uses tokens — acceptable exception for semantic context color)
- Any `border-radius: 20px` or `border-radius: 24px` on buttons — pill shape is only permitted for `.btn-full-pill` utility (FAB, filter pills). Regular buttons use the matrix above.

## .pib-card Density Rules

`.pib-card` is the canonical card primitive defined in `components/pib-cards.css`. All cards in the app must either use `.pib-card` or `.timeline-card` — no one-off card implementations in page CSS.

| Property | Value | Token |
|----------|-------|-------|
| Background | `--color-surface` | white |
| Border | `1px solid --color-border` | `--border-subtle` |
| Border radius | `--radius-lg` | 12px |
| Padding (default) | `--space-md` | 16px |
| Padding (compact variant) | 10px 12px | pages with dense lists only |
| Shadow (resting) | `--shadow-sm` | subtle |
| Shadow (hover) | `--shadow-md` | lift |
| Hover border | `--color-primary` | blue-500 accent |
| Hover transform | `translateY(-2px)` or `translateY(-3px)` | max −4px |
| Active/press transform | `scale(0.97)` | tactile |
| Gap between child elements | `--space-sm` (8px) | internal flex gap |

**Card title (`.pib-card-title`):** `--font-size-xl` (20px) / weight 700 / `--line-height-tight`.  
Note: current declaration in pib-cards.css uses `1.1rem` (17.6px) and `weight 800`. This spec normalises it to `--font-size-xl` (20px) / `--font-weight-bold` (700) to match the type scale. Executor must update pib-cards.css.

**Card body (`.pib-card-body`):** `--font-size-sm` (14px) / `--font-weight-normal` / `--line-height-normal` (1.5).  
Note: current declaration uses `0.9rem` (14.4px) — acceptable, but should be replaced with `--font-size-sm` token.

**Card meta (`.pib-card-meta`):** `--font-size-xs` (12px) / `--font-weight-semibold` (600).

**`.pib-card-date`:** `--font-size-xs` (12px) / `--font-weight-bold` (700) / uppercase / `--color-primary`.

**Accent strip rule:** Cards with left-side accent (schedules, spiritual items) use `border-left: 4px solid [semantic color]`. The strip width is locked at 4px. The `scale-card::before` pseudo-element in escalas.css uses 5px — normalise to 4px.

**Dashboard `.action-card` (lider.css):** These are intentionally different (gradient backgrounds, white text, 160px height for desktop). They are exempt from `.pib-card` density rules but must still conform to spacing tokens (`padding: var(--space-6)` on desktop, `var(--space-4)` on mobile ≤768px).

## Header Behaviour Rules

### `.page-sub-header` — Locked Behaviour Contract

| Context | Behaviour |
|---------|-----------|
| Dashboard pages (`admin/index.php`, `app/index.php`) | **Hide entirely** — user identity and theme toggle are accessible via sidebar (desktop) or bottom-nav profile item (mobile). Header adds no navigation value on the root dashboard. Apply `display: none` via a `.page-is-dashboard` body class or `.page-sub-header--hidden` modifier. |
| All internal pages (non-dashboard) | Show at 48px height (`page-sub-header-inner` height locked to 48px) |
| Mobile ≤1024px | 46px height (already declared in media query — preserve) |
| Desktop ≥1025px | 48px height |
| Subtitle row (`.page-sub-subtitle`) | Hidden on mobile ≤1024px (already declared — preserve) |
| User profile pill | Show only on desktop; on mobile ≤640px, pill collapses to avatar-only (already declared — preserve) |
| Dark mode | `background: rgba(10, 18, 32, 0.95)` (already declared — preserve) |

**Profile/theme in header:** The `.user-profile-pill` and `.header-action-btn` (bell, crown) live in the header and are the only persistent user-context controls. They must NOT be removed during harmonisation — only hidden on dashboard pages where the sidebar provides the same access.

**nav button sizing:** `.page-nav-btn` is 32×32px — below 44px touch target. This is acceptable because these are back/forward navigations adjacent to each other (compound touch region). Do not artificially inflate to 44px as it would break the compact header design. However, add `padding: 6px` to increase the clickable zone without changing visual size.

## Implementation Notes (Non-Blocking Recommendations)

1. **Execution order matters.** Remove the five `:root` override blocks FIRST (oracao, devocionais, leitura, evento-detalhe, avisos). Every other fix depends on the design-system cascade being clean.

2. **`.pib-card` adoption.** Several pages still use one-off card implementations (`.prayer-card`, `.devotional-card`, `.kpi-card` in relatorios.css). These do not need to be fully migrated to `.pib-card` in Phase 10 — just ensure they use `--radius-lg`, `--shadow-sm`, `--border-subtle` tokens instead of raw values.

3. **`page-sub-header` on dashboard.** The recommended implementation is to add a `page-is-dashboard` class to the `<body>` in `admin/index.php` and `app/index.php`, then add to `page-sub-header.css`: `body.page-is-dashboard .page-sub-header { display: none; }`. This is zero-risk and reversible.

4. **Visual audit sequence.** After CSS fixes, audit in this order: mobile 375px → desktop 1280px → dark mode mobile. Regression test: escalas listing, escala_detalhe (músico view), repertório, dashboard admin, devocional, oracao.

5. **Short-term alias removal.** `design-system.css` has two parallel naming systems (`--font-size-sm` and `--font-sm` / `--text-sm`). Phase 10 should standardise on the `--font-size-*` names (they match Tailwind conventions, are more descriptive). Do not remove `--font-sm` / `--text-sm` aliases yet — too many pages use them. Deprecation is a Phase 11 concern.
